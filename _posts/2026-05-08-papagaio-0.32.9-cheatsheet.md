Guia rápido para uso cotidiano do Papagaio.

## 1. Sintaxe de Padrões (Patterns)

Padrões são compostos por tokens literais e variáveis. O motor ignora espaços em branco horizontais entre tokens por padrão.

| Sintaxe | Descrição | Exemplo |
| :--- | :--- | :--- |
| `Literal` | Corresponde exatamente ao texto. | `hello` |
| `$var` | Captura texto até o próximo padrão. | `$name` |
| `${var}` | Delimita o nome da variável (evita ambiguidade). | `${id}suffix` |
| `$$` | Escapa o sigilo (ex: `$$` vira `$`). | `$$100` |
| `?` | Torna o token anterior opcional. | `$opt?` ou `maybe?` |
| `$` (final) | Colapsa espaços em branco seguintes. | `$var$` |

### Colapso de Espaço (`$`)
Adicionar um `$` ao final de uma variável ou literal faz com que o Papagaio consuma todos os espaços, abas e quebras de linha seguintes na entrada.
- `$pattern {$a$ $b} {$a/$b}` -> `hello   world` vira `hello/world`.

## 2. Modificadores de Variáveis

Modificadores restringem o que uma variável pode capturar.

### Tipos de Dados
- `$v$int`: Inteiros (positivos ou negativos).
- `$v$float`: Números decimais.
- `$v$number`: Qualquer número.
- `$v$percent`: Números com suporte a `%` e negativos.
- `$v$hex`: Valores hexadecimais (0-9, a-f, suporta prefixo `0x`).
- `$v$binary`: Valores binários (0, 1, suporta sufixo/prefixo `b`).

### Texto e Identificadores
- `$v$word`: Uma única palavra (sem espaços).
- `$v$identifier`: Identificador estilo C (letras, números, `_`).
- `$v$upper`: Texto em caixa alta.
- `$v$lower`: Texto em caixa baixa.
- `$v$capitalized`: Texto iniciando com maiúscula.
- `$v$path`: Caminhos de arquivo.

### Estruturas e Lógica
- `$v$block{open}{close}`: Captura tudo entre delimitadores (balanceado).
- `$v$aliases{a}{b}{c}`: Captura apenas se corresponder a um dos valores.
- `$v$regex{pattern}`: Captura usando uma expressão regular.
- `$v$group{subpattern}`: Agrupa uma sequência como uma única unidade.

### Filtros de String
- `$v$starts{pfx}`: Deve começar com `pfx`.
- `$v$ends{sfx}`: Deve terminar com `sfx`.
- `$v$prefix{pfx}`: Captura se começar com `pfx` (o prefixo é parte da captura).
- `$v$suffix{sfx}`: Captura se terminar com `sfx`.
- `$v$infix{sub}`: Captura se contiver `sub` no meio (não nas pontas).
- `$v$includes{sub}`: Captura se contiver `sub` em qualquer lugar.

## 3. Diretivas Embutidas (Built-ins)

Directivas de controle do motor.

| Diretiva | Descrição |
| :--- | :--- |
| `$pattern {pat} {rep}` | Define uma regra de substituição global. |
| `$from {val}` | Atribui `val` a uma variável `$NAME`. |
| `$priority$N {block}` | Define prioridade de execução (0 é a maior). |
| `$changesymbols{s}{o}{c}{q}` | Muda sigilo, delimitadores e marcador opcional. |
| `$document` | Injeta o conteúdo atual (atalho para `$document$current`). |
| `$document$original` | Injeta o conteúdo original (sem processamento). |
| `$document$current` | Injeta o estado atual do documento (pode ser recursivo). |

### Atribuição Dinâmica (`$from`)
Define uma variável que pode ser usada em qualquer lugar posterior do documento ou em outros padrões.
- `$NAME$from{Alice}` -> Define `$NAME` como `Alice` e remove a diretiva do output.

### Sistema de Prioridade (`$priority`)
Controla a ordem de execução. Útil para definir padrões antes de usá-los.
- `$priority$max`: Prioridade absoluta mais alta.
- `$priority$min`: Prioridade absoluta mais baixa.

## 4. Plugins e Extensibilidade

### Carregamento
- `$wasm{path.wasm}`: Carrega um plugin WebAssembly (registra comandos `papagaio_*`).
- `$wat{source}`: Compila e carrega código WAT (WebAssembly Text) inline.
- `$file{path}`: Injeta o conteúdo de um arquivo externo.

### Uso de Comandos de Plugin
Se um plugin exporta `papagaio_greet`, você o chama como:
- `$greet{arg1}{arg2}`

## 5. Argumentos de Linha de Comando (CLI)

O Papagaio expande argumentos passados na execução: `papagaio input.txt key=value extra`.

| Variável | Descrição |
| :--- | :--- |
| `$args$0` | Nome do script/arquivo de entrada (argv[1]). |
| `$args$1`, `$args$2`... | Argumentos posicionais subsequentes. |
| `$args$count` | Total de argumentos (excluindo o executável). |
| `$args` ou `$args$all` | Todos os argumentos extras (depois do script). |
| `$key` ou `$args$key` | Valor de um argumento `key=value` (Suporte Direto). |

### Suporte Direto a Variáveis
Se você passar `versao=1.0` na CLI, pode usar `$versao` diretamente no texto, desde que não haja um comando com o mesmo nome.

## 7. Desenvolvimento de Plugins (C)

Para criar um comando `$hello{name}` em C:

```c
#include "lib.c" // SDK do Papagaio (opcional, para malloc/sprintf)

char* papagaio_hello(int argc, char **argv) {
    // Papagaio adiciona automaticamente:
    // __attribute__((visibility("default")))
    if (argc < 1) return "Hello!";
    char *buf = malloc(strlen(argv[0]) + 10);
    sprintf(buf, "Hello, %s!", argv[0]);
    return buf;
}
```

Compilação recomendada:
```sh
# 1. Preprocessar
papagaio plugin.c > plugin.ready.c
# 2. Compilar para WASM
clang --target=wasm32-unknown-unknown -ffreestanding -nostdlib \
      -Wl,--no-entry,--export-dynamic plugin.ready.c -o plugin.wasm
```
