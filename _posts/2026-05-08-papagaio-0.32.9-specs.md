O Papagaio é uma engine de processamento de texto embarcável. Ele foi projetado para ser altamente modular e **script-agnostic**, permitindo que o core de **pattern matching** seja usado isoladamente ou estendido através de plugins WebAssembly (Wasm).

# Features

- **Lightweight Core**: Engine C eficiente para pattern matching e transformação.
- **Pattern Matching**: Sistema de captura poderoso com modificadores integrados e customizados.
- **Wasm Plugins**: Arquitetura de plugins altamente segura e zero-dependency via runtime `wasm3` embutido.
- **Configurable Delimiters**: Redefina sigils, delimitadores e markers em tempo de execução.
- **Language Bindings**: Uso nativo em C e Node.js/WebAssembly.

# Quick Start

## Interface de Linha de Comando (CLI)
```sh
# Processar com padrões definidos no arquivo ou passados via -e
papagaio -e '$pattern {hello $w} {Hi $w}' input.txt
```

## API C
```c
Papagaio *ctx = papagaio_open();

char *out = papagaio_process_text(ctx, input_text, strlen(input_text));
printf("%s", out);
free(out);
papagaio_close(ctx);
```

## JavaScript / WASM (Node.js)
```javascript
import Papagaio from './papagaio.js';

const p = new Papagaio();
await p.init();
p.registerCommand("mycmd", (name, ...args) => `Result: ${args[0]}`);

console.log(p.process('$mycmd{Hello}')); // Output: Result: Hello
```

# Pattern Syntax

Os padrões são compostos por tokens separados por whitespace. O engine utiliza uma estratégia de **flex-matching** que ignora automaticamente espaços em branco horizontais entre variáveis.

- **Literal**: Corresponde ao texto exato.
- **Variable**: `$nome` (captura uma sequência até o próximo match do padrão).
- **Optional**: `$nome?` ou `literal?` (o marker é configurável, ex: `MAYBE`, via `$changesymbols`).
- **Escaping**: Use `$$` para corresponder a um literal `$`.

## Modifiers
Os modificadores especificam o tipo de dado ou restrições de um match:
- **Numbers**: `$var$int`, `$var$float`, `$var$number`
- **Casing**: `$var$upper`, `$var$lower`, `$var$capitalized`
- **Formats**: `$var$word`, `$var$identifier`, `$var$hex`, `$var$path`, `$var$binary`, `$var$percent`
- **Regex**: `$id$regex{[0-9]+}`
- **Block**: `$item$block{[}{]}` (captura tudo entre os delimitadores)
- **Aliases**: `$kind$aliases{cat}{dog}{bird}` (sintaxe multi-bloco).
- **Substrings**: `$var$starts{foo}`, `$var$ends{bar}`, `$var$prefix{p}`, `$var$suffix{s}`, `$var$infix{i}`, `$var$includes{x}`
- **Grouping**: `$item$group{subpattern}` (agrupamento recursivo, corresponde como uma unidade)
- **Optionality**: qualquer token (literal, variável ou grupo) pode ser tornado opcional adicionando `?` (ou um marker customizado como `MAYBE`).
- **Trailing Sigil (whitespace collapse)**: anexar um `$` (ou o sigil atual) diretamente após qualquer variável ou literal faz com que o matcher **consuma todo o whitespace seguinte** na entrada. Isso é útil quando o número de espaços entre tokens é variável:
  ```text
  $pattern {$a$ $b} {$a/$b}
  hello   world   → hello/world
  ```
  O `$` final após `$a` colapsa qualquer run de espaços/tabs/quebras de linha entre `$a$ e `$b`.

## Braced Variables

Quando o nome de uma variável capturada precisa ser seguido imediatamente por texto literal (ex: um sufixo), envolva o nome em `${...}` para evitar ambiguidade:

```text
$pattern {$id$word} {${id}x}
foo
```
*Output: `foox`* — sem chaves, `$idx` seria interpretado como uma única variável chamada `idx`.

A sintaxe de chaves pode ser usada em qualquer string de substituição:
```text
$pattern {$first $last} {Hello, ${first}!
}
John Doe
```
*Output: `Hello, John!`*

## Nesting
Os modificadores suportam nesting recursivo total:
```text
$pattern {$n$aliases{$x$int}{abc}} {VALUE: $n}
```

# Extensibilidade (Wasm Plugins)

O Papagaio segue uma arquitetura de plugins **Wasm-first**. Os recursos principais são limitados ao pattern matching e transformação, enquanto capacidades customizadas de processamento de texto são fornecidas por plugins WebAssembly.

## Built-in Operators
- **`$document`**: Injeta o estado atual do documento (alias para `$document$current`).
- **`$document$original`**: Injeta o texto de entrada inicial, não processado. Útil para referenciar a fonte mesmo após múltiplas transformações.
- **`$document$current`**: Injeta o estado atual do documento durante o pré-processamento.
- **`$wasm{path}`**: Carrega um plugin WebAssembly do filesystem (CLI apenas).
- **$file{path}**: Injeta o conteúdo de um arquivo do filesystem (CLI apenas).
- **`$wat{source}`**: Compila uma fonte WebAssembly Text Format (WAT) inline e registra todas as funções `papagaio_*` exportadas como comandos. Útil para embutir plugins leves sem um arquivo `.wasm` externo.
- **`$NAME$from{value}`**: Atribui dinamicamente um `value` processado a `$NAME`. A atribuição em si é suprimida da saída, e a variável torna-se disponível para substituição de match exato no restante do documento.

  ```text
  $NAME$from{Alice}
  Hello, $NAME!
  ```
  *Output: `Hello, Alice!`*

  ```text
  $wat{
    (module
      (func (export "papagaio_hello") (result i32)
        i32.const 42))
  }
  $hello
  ```

# CLI Argument Expansion

O Papagaio pode resolver argumentos de linha de comando diretamente em seus arquivos fonte. Isso é útil para passar configurações, flags ou metadados para o pipeline de processamento.

## Positional Arguments
O array `argv` é mapeado da seguinte forma (onde `argv[0]` é o nome do binário, invisível para o Papagaio):

| Variável | Valor |
|---|---|
| `$args$0` | `argv[1]` — o nome do arquivo de entrada/script |
| `$args$1`, `$args$2`, … | Argumentos posicionais subsequentes |
| `$args$count` | Número total de argumentos (exclui o nome do binário, `argv[0]`) |
| `$args$all` | Todos os argumentos extras a partir do índice 1 (após o script), unidos por espaços |

Se uma variável `$args$NAME` não for encontrada, ela é emitida **literalmente** (ex: `$args$missing` permanece como está).

## Named Variables (Overrides)
Argumentos no formato `key=value` são analisados automaticamente e podem ser acessados de duas maneiras:
1. **Explicit**: `$args$key`
2. **Direct**: `$key` (atalho para `$args$key`)

O acesso direto (`$key`) só será resolvido se `key` não conflitar com um comando registrado (como `$wasm`) ou uma diretiva built-in.

### Exemplo:
```sh
./papagaio input.c version=1.2.3 target=wasm -O3 > ready.c
# Then compile ready.c with clang to WASM
```
Dentro de `input.c`:
```c
const char *v = "$version"; // "1.2.3"
const char *t = "$target";  // "wasm"
const char *f = "$args$1";  // "-O3"
```

# Customização Dinâmica

Você pode redefinir os símbolos de sintaxe do engine em tempo de execução usando a diretiva atômica **`$changesymbols`**.
{% raw %}
## `$changesymbols{sigil}{open}{close}{optional}`
Default: `$changesymbols{$}{{} }{}}{?}`
{% endraw %}
Exemplo:
```text
$changesymbols{@}{<}{>}{!} @pattern <@n!> <ID: @n> [x] [y]
```
Isso muda o sigil para `@`, os delimitadores para `< >` e o optional marker para `!`. As diretivas de preprocessor (como o próprio `$changesymbols`) sempre usam os estáveis `$` e `{}` para permanecerem funcionais.

# Recursive Priority System

O Papagaio permite controlar a ordem de execução e side-effects (como definições de padrões ou carregamento de WASM) usando a diretiva **`$priority$N`**.

- **`$priority$0{...}`**: Prioridade máxima.
- **`$priority$max{...}`**: Alias para `INT_MIN + 1` (Absolute highest priority).
- **`$priority$min{...}`**: Alias para `INT_MAX - 1` (Absolute lowest priority).
- **`$priority$1`, `$priority$2`, ...**: Prioridades sucessivamente menores.
- **Recursive Evaluation**: Blocos com maior prioridade numérica (valor menor) são totalmente processados — incluindo seus próprios padrões e comandos aninhados — antes de qualquer bloco de menor prioridade, independentemente de sua posição física no arquivo.
- **Unspecified Priority**: Qualquer texto não envolto em um bloco `$priority` é tratado como prioridade `INT_MAX - 1` (processado por último).

### Exemplo:
```text
$priority$1{ Result: A }
$priority$max{ $pattern{A}{OK} }
```
*Output: `Result: OK`* — embora `A` seja usado antes de ser definido na fonte, o bloco `$priority$max` garante que a definição do padrão ocorra primeiro.

# Dynamic Variable Assignment ($from)

O operador **`$from`** permite capturar conteúdo processado e atribuí-lo a uma variável em tempo de execução. Isso torna o Papagaio um processador stateful, onde variáveis podem ser definidas, redefinidas e encadeadas.

## Syntax
`$NAME$from{...content...}`

1. **Recursive Processing**: O `content` é totalmente processado (patterns, comandos WASM, outras atribuições) antes de ser armazenado.
2. **Immediate Registration**: A variável é registrada como uma regra de match exato assim que é analisada. Isso permite **chained assignments**.
3. **Output Suppression**: Toda a diretiva `$from` é removida do texto de saída.

## Exemplos

### Chained Assignments
Variáveis podem depender de variáveis definidas anteriormente no mesmo documento:
```text
$A$from{1}
$B$from{$A$A}
Count: $B
```
*Output: `Count: 11`*

### Nested Assignments
Você pode definir variáveis internas ao definir um bloco maior:
```text
$GREET$from{
  $USER$from{Alice}
  Hello, $USER!
}
$GREET
```
*Output: `Hello, Alice!`*

### Interaction with Patterns
Atribuições podem ser usadas para gerar dinamicamente chaves de padrões ou substituições:
```text
$KEY$from{FOO}
$pattern{$KEY}{BAR}
Result: FOO
```
*Output: `Result: BAR`*

# Plugin Development

O Papagaio apresenta um sistema de plugins Wasm moderno e sem atrito. Você pode escrever plugins em C padrão usando convenções de nomes simples e compilá-los em módulos WebAssembly zero-dependency usando o `papagaio` como preprocessor e o `clang` como compilador WASM.

## 1. Escreva seu plugin
Crie um arquivo chamado `greet.c`:
```c
// Funções que começam com 'papagaio_' são registradas automaticamente como comandos
char* papagaio_greet(int argc, char **argv)
{
    if (argc < 1) return "Hello, Stranger!";
    return argv[0]; // retorna o primeiro argumento
}
```

Para usar o Papagaio Wasm SDK (`lib.c`), copie-o de `examples/lib.c` para o seu projeto e inclua-o explicitamente:
```c
#include "lib.c"

char* papagaio_greet(int argc, char **argv)
{
    if (argc < 1) return "Hello, Stranger!";
    
    // lib.c fornece funções C padrão como malloc e sprintf
    char *res = (char*)malloc(strlen(argv[0]) + 16);
    sprintf(res, "Hello, %s!", argv[0]);
    
    return res;
}
```

## 2. Compile para WebAssembly
Para compilar seu plugin, primeiro processe-o com o `papagaio` e depois compile o resultado com o `clang` usando as seguintes flags:

```sh
# 1. Preprocess com Papagaio
./papagaio greet.c > greet.ready.c

# 2. Compile para WASM com Clang
clang --target=wasm32-unknown-unknown -ffreestanding -nostdlib -fvisibility=hidden \
      -Wl,--no-entry,--export-dynamic,--allow-undefined \
      greet.ready.c -o greet.wasm
```

Or simplesmente use o Makefile fornecido no diretório `examples/`:
```sh
cd examples && make
```

## 3. Use no Papagaio
Carregar o arquivo Wasm registra automaticamente todos os comandos exportados.
```text
$wasm{greet.wasm}
$greet{Papagaio}
```
*Output: Hello, Papagaio!*

## Wasm SDK (lib.c)
O Wasm SDK vive em `examples/lib.c` dentro do repositório. Ele fornece uma biblioteca padrão C selecionada e zero-dependency para WebAssembly, incluindo:
- **Memory Management**: `malloc`, `free`, `realloc`
- **String Processing**: `strlen`, `strcpy`, `sprintf`, `strrev`, etc.
- **Formatted I/O**: `printf`, `snprintf`, `sscanf`
- **Standard Math**: `sin`, `cos`, `pow`, etc.

# System Limits

| Feature | Limit | Detail / Technical Rationale |
|---|---|---|
| **Symbol Length** | 15 characters | Sigils, delimiters (`{`, `}`) e markers são armazenados em buffers fixos de 16 bytes. |
| **String Size** | Unlimited | Todos os buffers internos (`StrBuf`) usam `realloc` dinâmico. Limitado apenas pela RAM disponível. |
| **Pattern Count** | Unlimited | Regras registradas são armazenadas em um array dinâmico. |
| **Priority Range** | `INT_MIN` a `INT_MAX` | Prioridades são tratadas como inteiros sinalizados padrão. |
| **Recursion Depth** | Stack-limited | Padrões profundamente aninhados ou blocos de prioridade são processados recursivamente. |
| **Wasm Memory** | 1 MB (default) | O runtime `wasm3` padrão é inicializado com 1MB. |
| **Wasm Arguments** | ~64 KB | Argumentos são mapeados no espaço de memória Wasm começando no offset 4096. |
